# 객체에 대한 이해
## 캡슐화
### 캡슐화는 객체를 캡슐로 싸서 그 내부를 볼 수 없게 하고 외부 접근으로부터 보호한다.
## C++ 객체
### C++ 객체는 멤버 변수와 멤버 함수로 구성되며, 멤버 변수는 객체의 상태를, 멤버 함수는 객체의 행동을 표현한다.
### C++ 클래스는 객체를 정의하는 틀 혹은 설계도로서 사용하는 클래스로 새로운 데이터 타입을 선언.
### C++ 객체는 실행 중에 생성되어 존재하므로 실체라고도 부른다.

# C++ 클래스 만들기
### class 키워드를 이용하여 선언. 클래스 선언 뒤에는 반드시 세미콜론(;)
### 클래스는 선언부와 구현부로 나눠 작성하는 것이 원칙. 클래스 구현부는 클래스 선언부에 선언된 함수들의 코드를 구현하는 부분
### 객체의 멤버 접근은 '객체.멤버' 형식을 사용

# 생성자
### 생성자는 클래스의 이름과 동일한 멤버 함수로서 객체가 생성될 때 한 번만 실행되는 특별한 멤버이다.
### 생성자는 객체의 초기화에 사용, 리턴 타입 없고 중복 가능
### 기본 생성자는 디폴트 생성자로도 불리며 매개 변수 없는 생성자이다. 아무 생성자도 선언되지 않으면 컴파일러가 기본 생성자를 자동 삽입. 기본생성자는 단순 리턴

# 소멸자
### 소멸자는 객체 소멸시 실행되는 멤버 함수로 클래스 이름 앞에 ~를 붙인 이름으로 선언.
### 소멸자는 매개 변수를 가지지 않고 리턴 타입도 없으며, 중복 불가능. 객체는 생성된 반대순 소멸

# 접근 지정
### 객체를 캡슐화 함에 따라 외부에서 접근 가능한 멤버와 접근 불가한 멤버를 선언하는 지시이다.
### 멤버 접근 지정 종류는 총 3가지 private(비공개), public(공개), protected(보호)
### private 멤버는 클래스 내부 함수들만 접근 가능
### public 멤버는 클래스 내외 모든 함수들이 접근 가능
### protected 멤버는 자식 클래스에서만 접근 가능

# 인라인 함수
### 함수에 inline 키워드를 붙여 인라인 선언시, 컴파일러는 인라인 함수 코드를 함수 호출 하는 곳에 확장. 따라서 함수 호출이 일어나지 않고 오버헤드를 제거하여 실행 속도를 높임.
### 인라인은 클래스 멤버 함수, 외부 함수 모두 가능하며, 클래스 내 선언부에 작성된 함수는 컴파일러에 의해 자동 인라인 처리

# C++ 구조체
### C언어와의 호환성을 위해 지원
### C++ 구조체는 디폴트 접근 지정이 public이라는 점을 제외하고, 멤버 함수, 멤버 변수, 상속 ,접근 지정 등 클래스의 기능과 동일

### main() 등과 같은 외부 함수는 따로 cpp 파일에 나눠 작성하는 것이 바람직
